## 8-Phase Workflow

| Phase | Name | Purpose |
|-------|------|---------|
| 0 | Pre-flight | Audits codebase, creates INVENTORY.md |
| 1 | Architect | Creates PLAN.md with refactoring strategy |
| 2 | Refactor | Executes plan on dedicated branch |
| 3 | Cleanup | Removes dead code with user confirmation |
| 4 | CLAUDE.md | Generates AI context guide |
| 5 | README.md | Human-facing docs with Mermaid diagrams |
| 6 | Tests | Scaffolds test infrastructure |
| 7 | Validation | Before/after comparison report |

## When to Use
- When you take over an existing client project
- When a codebase has grown messy over time
- When onboarding a new AI agent to an unfamiliar repo
